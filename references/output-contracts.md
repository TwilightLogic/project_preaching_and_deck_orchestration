# Output Contracts

Use these stage outputs exactly. Keep headings stable so later stages can build on earlier ones without reformatting.

## Minimum Useful Input

Aim to collect:
- what the project is
- who the audience is
- what decision or reaction the presentation should drive
- what proof already exists
- where the evidence is weak

If some of this is missing, infer cautiously and list the gap explicitly in Stage A rather than blocking.

Global rule:
- In collaborative mode, default to one stage per response.
- In end-to-end mode, keep the stage order intact and preserve the heading and field order for every stage.
- In end-to-end mode, compress each stage rather than skipping it.

## Stage A: Project Understanding Brief

Include:
- Project one-sentence summary
- Project goals
- Problem being solved
- Core capabilities
- Target users, customers, or stakeholders
- Differentiated value
- Known strong evidence
- Weak evidence or assumptions
- Risks and uncertainties
- Missing information

Rules:
- Do not jump into slide structure before this brief is complete.
- Make the confidence level of each conclusion clear.
- If materials are thin, include the best current framing and a short note on what would strengthen it.

## Stage B: Argumentation Map

Create 3 to 7 core claims. For each claim, include:
- Claim
- Evidence
- Reasoning
- Evidence strength
- Potential challenge
- Recommended wording

Rules:
- Separate evidence from reasoning explicitly.
- Mark evidence as direct, indirect, assumption, or missing information.
- Keep the map usable for later stages. Each claim should be short enough to turn into a deck message.

## Stage C: Narrative Comparison

Compare at least three narrative options. For each option, include:
- Narrative spine
- Why it fits
- Risk or weakness

End with:
- Recommended narrative
- Why it is the best fit for the audience and proof profile

Rules:
- Prefer the narrative that makes the proof easiest to believe, not just the one that sounds the most exciting.

## Stage D: Deck Blueprint

For every slide, include:
- Slide number
- Title
- Key message
- Suggested content
- Speaker intent
- Transition

Rules:
- Keep one core idea per slide.
- Prefer visuals, comparisons, flows, and proof points over dense paragraphs.
- Default to roughly 6 to 12 slides unless the user scope clearly requires more.

## Stage E: Page-by-Page Talk Script

For every slide, include:
- Page title
- Spoken script
- Suggested duration
- Emphasis line
- Optional visual cue

Rules:
- Write natural spoken delivery, not slide text read aloud.
- Match the tone to the page role: opening, turn, proof, climax, or close.
- Keep each page script concise enough to rehearse and expand in live delivery.
